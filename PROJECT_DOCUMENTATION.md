## 1. Problem Definition and Product Goals

### The Problem
Modern home cooks face a common frustration: standing in front of the refrigerator with various ingredients but no idea what to cook. Traditional solutions involve:
- Manually searching through recipe websites
- Scrolling through irrelevant ads and personal stories
- Guessing which recipes will work with available ingredients
- Wasting time and often giving up, leading to food waste or ordering takeout

This problem results in:
- **Food waste** - ingredients expire before being used
- **Decision fatigue** - too many options, too little guidance
- **Time waste** - searching takes longer than cooking
- **Frustration** - hunger + indecision = poor experience

### The Solution
"The Improvising Chef" is an AI-powered web application that instantly transforms available ingredients into complete, practical recipes. Users simply:
1. Enter their ingredients (or upload a photo)
2. Click one button
3. Receive 3 complete recipes with detailed instructions

### Product Goals
‚úÖ **Primary Goal:** Reduce the time from "I have ingredients" to "I know what to cook" from 15-30 minutes to under 30 seconds
‚úÖ **Secondary Goal:** Reduce food waste by helping users utilize what they already have
‚úÖ **User Experience Goal:** Create a clean, simple interface with zero friction
‚úÖ **Technical Goal:** Demonstrate multi-modal AI integration (text generation, image analysis)

### Target Users
- Busy professionals who want quick meal ideas
- Students with limited ingredients and budget
- Home cooks looking to reduce food waste
- Anyone experiencing decision fatigue around cooking

## 6. Challenges and Solutions

### Technical Challenges

#### Challenge 1: Environment Variable Loading
**Problem:** `.env` file created but API key not recognized

**Error:**
```
OpenAIError: Missing credentials. Please pass an apiKey,
or set the OPENAI_API_KEY environment variable.
```

**Root Cause:** `dotenv.config()` called AFTER requiring files that need env vars

**Solution:**
```javascript
// BEFORE (Wrong)
const express = require('express');
const recipeRoutes = require('./routes/recipeRoutes'); // Uses API key!
dotenv.config(); // Too late!

// AFTER (Correct)
const dotenv = require('dotenv');
dotenv.config(); // First!
const express = require('express');
const recipeRoutes = require('./routes/recipeRoutes'); // Now it works
```

**Lesson Learned:** Environment variables must load before any code that uses them

#### Challenge 2: JSON Parsing from AI
**Problem:** GPT-4o-mini wraps JSON in markdown code blocks

**Error:**
```
SyntaxError: Unexpected token '`', "```json [...]" is not valid JSON
```

**AI Output:**
```json
```json
[
  { "name": "Recipe", ... }
]
```
```

**Solution:**
```javascript
let response = completion.choices[0].message.content;
// Strip markdown code blocks
response = response.replace(/```json\s*/g, '')
                   .replace(/```\s*/g, '')
                   .trim();
const recipes = JSON.parse(response);
```

**Lesson Learned:** AI outputs may need preprocessing before parsing

#### Challenge 3: API Billing Limits
**Problem:** DALL-E 3 hit billing hard limit

**Error:**
```
Error generating image: 400 Billing hard limit has been reached
```

**Impact:** No dish images generated

**Solutions Considered:**
1. ‚ùå Use cheaper image API ‚Üí Quality too low
2. ‚ùå Use placeholder images ‚Üí Not real
3. ‚úÖ Disable images, focus on recipes ‚Üí App still valuable

**Implementation:**
```javascript
// Simple: Just return null for images
const recipesWithImages = recipes.map(recipe => ({
  ...recipe,
  imageUrl: null
}));
```

**Lesson Learned:** Build graceful degradation from the start

#### Challenge 4: Model Availability
**Problem:** "gpt-4" not available on all API accounts

**Solution:** Switch to "gpt-4o-mini"

**Benefits:**
- ‚úÖ More accessible (available to everyone)
- ‚úÖ 99% cheaper ($0.15 vs $30 per 1M tokens)
- ‚úÖ Faster response times
- ‚úÖ Still excellent quality for recipes

**Lesson Learned:** Choose models based on accessibility + cost, not just capability

#### Challenge 5: Layout Centering
**Problem:** All content aligned to left side of screen

**Root Cause:** Conflicting CSS from Vite template

**Culprit:**
```css
/* index.css from Vite template */
body {
  display: flex;
  place-items: center; /* This caused issues */
}
```

**Solution:**
```css
/* Remove conflicting styles */
body {
  margin: 0;
  min-width: 320px;
  min-height: 100vh;
}

/* Add proper centering to app container */
.app-main {
  max-width: 1200px;
  width: 100%;
  margin: 0 auto;
  display: flex;
  flex-direction: column;
  align-items: center;
}
```

**Lesson Learned:** Check template CSS for conflicts
