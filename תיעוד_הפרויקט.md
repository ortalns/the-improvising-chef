## 1. הגדרת הבעיה ומטרות המוצר

### הבעיה
מבשלים ביתיים מודרניים מתמודדים עם תסכול נפוץ: עומדים מול המקרר עם מרכיבים שונים אך ללא מושג מה להכין. הפתרונות המסורתיים כוללים:
- חיפוש ידני באתרי מתכונים
- גלילה דרך פרסומות וסיפורים אישיים לא רלוונטיים
- ניחוש אילו מתכונים יעבדו עם המרכיבים הזמינים
- בזבוז זמן ולעיתים קרובות ויתור, שמוביל לבזבוז מזון או להזמנת משלוח

בעיה זו מביאה ל:
- **בזבוז מזון** - מרכיבים מתקלקלים לפני השימוש
- **עייפות החלטות** - יותר מדי אפשרויות, מעט מדי הנחיה
- **בזבוז זמן** - החיפוש לוקח יותר זמן מהבישול
- **תסכול** - רעב + חוסר החלטיות = חוויה גרועה

### הפתרון
"השף המאלתר" היא אפליקציית ווב מונעת בינה מלאכותית שהופכת באופן מיידי מרכיבים זמינים למתכונים שלמים ומעשיים. משתמשים פשוט:
1. מזינים את המרכיבים שלהם (או מעלים תמונה)
2. לוחצים על כפתור אחד
3. מקבלים 3 מתכונים שלמים עם הוראות מפורטות

### מטרות המוצר
✅ **מטרה ראשית:** לקצר את הזמן מ"יש לי מרכיבים" ל"אני יודע מה להכין" מ-15-30 דקות לפחות מ-30 שניות
✅ **מטרה משנית:** להפחית בזבוז מזון על ידי עזרה למשתמשים לנצל את מה שיש להם
✅ **מטרת חוויית משתמש:** ליצור ממשק נקי ופשוט ללא חיכוכים
✅ **מטרה טכנית:** להדגים שילוב בינה מלאכותית רב-מודאלית (יצירת טקסט, ניתוח תמונות)

### קהל יעד
- אנשי מקצוע עסוקים שרוצים רעיונות ארוחה מהירים
- סטודנטים עם מרכיבים ותקציב מוגבלים
- מבשלים ביתיים שמחפשים להפחית בזבוז מזון
- כל מי שחווה עייפות החלטות סביב בישול

### טכנולוגיות Frontend

#### 1. React 18
**מטרה:** מסגרת UI לבניית רכיבים אינטראקטיביים

**למה React:**
- ארכיטקטורה מבוססת רכיבים (ניתנת לשימוש חוזר, ניתנת לתחזוקה)

**מושגי מפתח בשימוש:**
- רכיבים פונקציונליים עם hooks
- useState לניהול state
- טיפול באירועים ו-props
- רינדור מותנה

#### 2. Vite
**מטרה:** כלי בנייה ושרת פיתוח

**למה Vite:**
- Hot module replacement (HMR) מהיר במיוחד
- בניות ייצור מותאמות
- תמיכה במודולי ES מודרניים
- אפס תצורה נדרשת

#### 3. Axios
**מטרה:** לקוח HTTP לבקשות API

**למה Axios:**
- API נקי מבוסס promises
- ניתוח JSON אוטומטי
- תמיכה ב-FormData להעלאות קבצים
- טיפול בשגיאות טוב יותר מ-fetch

#### 4. CSS3
**מטרה:** עיצוב ועיצוב רספונסיבי

**תכונות בשימוש:**
- Flexbox לפריסה
- CSS Grid לכרטיסי מתכונים
- אנימציות ומעברי CSS
- שאילתות מדיה לרספונסיביות

## 6. אתגרים ופתרונות

### אתגרים טכניים

#### אתגר 1: טעינת משתני סביבה
**בעיה:** קובץ `.env` נוצר אך מפתח API לא מזוהה

**שגיאה:**
```
OpenAIError: Missing credentials. Please pass an apiKey,
or set the OPENAI_API_KEY environment variable.
```

**סיבת שורש:** `dotenv.config()` נקרא אחרי requiring קבצים שצריכים משתני env

**פתרון:**
```javascript
// לפני (שגוי)
const express = require('express');
const recipeRoutes = require('./routes/recipeRoutes'); // משתמש במפתח API!
dotenv.config(); // מאוחר מדי!

// אחרי (נכון)
const dotenv = require('dotenv');
dotenv.config(); // ראשון!
const express = require('express');
const recipeRoutes = require('./routes/recipeRoutes'); // עכשיו זה עובד
```

**לקח שנלמד:** משתני סביבה חייבים להיטען לפני כל קוד שמשתמש בהם

#### אתגר 2: ניתוח JSON מ-AI
**בעיה:** GPT-4o-mini עוטף JSON בבלוקי קוד markdown

**שגיאה:**
```
SyntaxError: Unexpected token '`', "```json [...]" is not valid JSON
```

**פלט AI:**
```json
```json
[
  { "name": "מתכון", ... }
]
```
```

**פתרון:**
```javascript
let response = completion.choices[0].message.content;
// הסר בלוקי קוד markdown
response = response.replace(/```json\s*/g, '')
                   .replace(/```\s*/g, '')
                   .trim();
const recipes = JSON.parse(response);
```

**לקח שנלמד:** פלטי AI עשויים לדרוש עיבוד מקדים לפני ניתוח

#### אתגר 3: מגבלות חיוב API
**בעיה:** DALL-E 3 הגיע למגבלת חיוב קשה

**שגיאה:**
```
Error generating image: 400 Billing hard limit has been reached
```

**השפעה:** לא נוצרו תמונות מנות

**פתרונות שנשקלו:**
1. ❌ שימוש ב-API תמונות זול יותר → איכות נמוכה מדי
2. ❌ שימוש בתמונות placeholder → לא אמיתי
3. ✅ השבתת תמונות, התמקדות במתכונים → האפליקציה עדיין בעלת ערך

**יישום:**
```javascript
// פשוט: פשוט החזר null לתמונות
const recipesWithImages = recipes.map(recipe => ({
  ...recipe,
